Data Collection and Annotation

Building Your Dataset
The Foundation of Any CV System

Garbage in, garbage out — why data quality matters more than model complexity

You've picked the perfect model architecture, but your training data consists of blurry images taken at night with a potato camera. No amount of tuning will save you. In this lesson, we'll learn how to collect and annotate data that actually works.

The Data Lifecycle

Before you write a single line of training code, you need to think about data. The data lifecycle for a computer vision project typically looks like this:

  1. Define Requirements: What classes do you need? What variations (lighting, angle, occlusion) must be handled?
  2. Source Data: Collect existing datasets, capture new data, or use synthetic generation
  3. Store and Organize: Set up a coherent directory structure and naming convention
  4. Annotate: Label objects of interest with bounding boxes, polygons, or keypoints
  5. Validate: Check annotation quality and inter-annotator agreement
  6. Split: Divide into training, validation, and test sets
  7. Augment: Apply transformations to increase dataset size and robustness

Data Collection Strategies

Depending on your use case, you might use one or more of these approaches:

  • Public Datasets: Start with COCO, ImageNet, OpenImages, or domain-specific datasets
  • Web Scraping: Collect images from search engines or social media (respecting copyright and privacy)
  • Synthetic Data: Use rendering engines like Blender or Unity to generate perfectly labeled data
  • Active Learning: Have the model identify uncertain samples for human annotation
  • Simulation: Use tools like CARLA for autonomous driving or Isaac Sim for robotics

Annotation Types and Tools

The type of annotation you need depends on your task:

Bounding Boxes

Rectangles around objects - fastest to annotate, used for object detection

Polygons/Segmentation Masks

Precise outlines - used for instance and semantic segmentation

Keypoints

Specific points on objects - used for pose estimation

Cuboids

3D bounding boxes - used for depth estimation and 3D object detection

Popular Annotation Tools

  • LabelImg: Simple, open-source tool for bounding boxes
  • LabelMe: More advanced, supports polygons and keypoints
  • CVAT: Web-based, collaborative annotation platform
  • Roboflow: End-to-end dataset management with augmentation
  • Scale AI: Professional annotation service

Data Quality Checklist

Before moving on, ask yourself these questions:

  1. Do I have enough samples per class (at least 1000 for deep learning)?
  2. Is my data representative of real-world conditions (lighting, weather, backgrounds)?
  3. Are my annotations consistent and accurate?
  4. Have I checked for class imbalance?
  5. Is my data properly split (no leakage between train/val/test)?
