Deployment to Embedded Systems

Getting Your Model on Hardware
From Framework to Embedded Runtime

Understanding the deployment process for Orin NX and QCS8550

You've trained and optimized your model, but now comes the tricky part: actually getting it to run on your target embedded device. Deployment involves more than just copying files - you need to consider runtime compatibility, hardware acceleration, and system integration.

The Deployment Pipeline

Deploying a CV model to embedded systems typically involves these steps:

  1. Model Export: Convert from training framework (PyTorch/TensorFlow) to deployment format
  2. Hardware Optimization: Use vendor-specific tools to optimize for target accelerator
  3. Runtime Integration: Embed the model in your application code
  4. System Testing: Validate performance, accuracy, and resource usage on target hardware
  5. Iteration: Go back and optimize further based on real-world measurements

NVIDIA Orin NX Deployment

For the Orin NX (Jetson platform), the deployment workflow looks like this:

  • Framework: Start with PyTorch model
  • Export: Convert to ONNX or TorchScript
  • Optimization: Use TensorRT for GPU acceleration
  • Runtime: Deploy with TensorRT, ONNX Runtime, or PyTorch C++/Python
  • Tools: JetPack SDK includes CUDA, cuDNN, TensorRT, and multimedia APIs

Orin NX Specific Tips

  • Leverage CUDA cores and Tensor cores for acceleration
  • Use shared memory (CPU-GPU) efficiently to avoid bottlenecks
  • Consider power modes - MAXN for performance, 10W/15W for efficiency
  • Use Jetson Multimedia API (JMAPI) for camera input if needed
  • Monitor with tegrastats for GPU/CPU usage and power consumption

Qualcomm QCS8550 Deployment

For the QCS8550 (Qualcomm platform), the deployment workflow is:

  • Framework: Start with PyTorch or TensorFlow model
  • Export: Convert to ONNX (preferred) or TensorFlow Lite
  • Optimization: Use Qualcomm AI Engine Direct SDK
  • Runtime: Deploy with Qualcomm Neural Processing SDK
  • Tools: QCS SDK includes Hexagon NN libraries, ADSP tools, and camera drivers

QCS8550 Specific Tips

  • Hexagon NPU is optimized for INT8 operations - prioritize INT8 quantization
  • Use Hexagon Vector Extensions (HVX) for efficient vector operations
  • Leverage the Adreno GPU for graphics or CV tasks that prefer floating point
  • Use the Qualcomm Sensor SDK for integrated sensor processing
  • Consider the Hexagon Tensor Accelerator (HTA) for tensor operations

Cross-Platform Considerations

If you need to deploy to both platforms:

  1. ONNX as Intermediate: Use ONNX as a common format that both platforms can optimize
  2. Abstract Hardware Layer: Design your inference wrapper to plug in different backends
  3. Benchmark Early: Test on both platforms to understand performance characteristics
  4. Consider Power vs Performance: QCS8550 often better for battery-powered; Orin NX for maximum performance

Deployment Checklist

Before declaring deployment complete:

  • Model loads successfully on target hardware
  • Inference produces expected outputs (numerical parity with training framework)
  • Latency meets real-time requirements (measure end-to-end including preprocessing)
  • Memory usage is within system limits
  • Power consumption is within budget
  • Thermal performance is sustainable for expected runtime
