Model Optimization and Quantization

Making Your Model Fast and Small
Techniques for Edge Deployment

From pruning to quantization - optimizing models for embedded systems

You've trained a model that works great on your desktop GPU, but when you try to run it on an Orin NX or QCS8550, it's too slow or uses too much memory. In this lesson, we'll learn how to optimize and compress models without sacrificing too much accuracy.

Why Optimize for Edge?

Embedded systems have strict constraints:

  • Limited Compute: Less GPU/NPU power than desktop cards
  • Memory Restrictions: Often 4-8GB RAM shared with OS
  • Power Budgets: Need to operate within watts, not hundreds of watts
  • Thermal Limits: No active cooling in many embedded designs
  • Real-time Requirements: Often need 30 FPS or higher for robotics

Optimization Techniques

Here are the main categories of model optimization:

Model Pruning

Remove unnecessary connections or channels in the network. Can be structured (removing whole filters) or unstructured (individual weights).

Quantization

Reduce numerical precision of weights and activations (e.g., FP32 → INT8 or FP16). Significantly reduces model size and speeds up inference on integer-capable hardware.

Knowledge Distillation

Train a smaller "student" network to mimic a larger "teacher" network. Often yields better small models than direct training.

Neural Architecture Search (NAS)

Automatically discover optimized architectures for specific hardware constraints.

Low-Rank Factorization

Decompose weight matrices into lower-rank approximations to reduce parameters.

Quantization Deep Dive

Quantization is particularly important for embedded deployment. Key concepts:

  • Post-Training Quantization (PTQ): Quantize a trained model without retraining. Fast but may lose accuracy.
  • Quantization-Aware Training (QAT): Simulate quantization during training to learn weights that are robust to quantization.
  • Dynamic vs Static Quantization: Dynamic quantizes activations at runtime; static uses pre-calculated scale factors.
  • Hardware Support: Orin NX supports INT8/FP16 on GPU; QCS8550 Hexagon NPU is optimized for INT8.

Tools and Frameworks

Popular tools for model optimization:

  • TensorRT: NVIDIA's optimizer for GPU deployment (great for Orin NX)
  • Qualcomm AI Engine Direct: SDK for optimizing models for Hexagon NPU
  • ONNX Runtime: Cross-platform inference engine with optimization options
  • OpenVINO: Intel's toolkit for optimizing and deploying models
  • TVM: Apache TVM for compiling models to various backends
  • PyTorch Mobile: For deploying PyTorch models on mobile/embedded

Practical Tips

  1. Start with PTQ to see baseline performance impact
  2. If accuracy drop is too high, try QAT
  3. Consider hybrid approaches (e.g., keep first and last layers in FP32)
  4. Profile your model to find bottlenecks before optimizing
  5. Test on actual hardware early - simulators can be misleading
