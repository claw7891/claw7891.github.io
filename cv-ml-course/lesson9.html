End-to-End Project and Best Practices

Bringing It All Together
From Concept to Deployed System

A complete walkthrough and best practices for successful CV projects

  • Goal: Automatically detect defects (cracks, hot spots, debris) in solar panels using drone imagery
  • Classes: Normal panel, crack, hot spot, debris, shading
  • Requirements: 95%+ accuracy, <1 second inference time, lightweight for drone mounting
  • Constraints: Must run on Orin NX with <10W power budget, handle varying lighting conditions

    • 2. Data Collection and Annotation

    • Data Sources: Combination of public solar panel datasets, synthetic renders, and collected drone footage
    • Variations: Different times of day, weather conditions, panel types, installation angles
    • Annotation: Bounding boxes for defects, with severity levels for cracks
    • Quality Control: Double-annotation for 20% of data, inter-annotator agreement >0.85 IoU

    3. Model Selection and Training

    • Architecture: YOLOv8s (good balance of speed and accuracy for small defect detection)
    • Transfer Learning: Started with COCO-pre-trained weights
    • Training: 100 epochs with cosine annealing scheduler, mixed precision
    • Augmentation: Random brightness/contrast, solarization, mosaic augmentation
    • Results: 0.92 mAP@0.5 on validation set

    4. Optimization and Quantization

    • Quantization: INT8 quantization using PyTorch QAT
    • Accuracy Impact: <1.5% mAP drop after quantization
    • Optimization: TensorRT FP16 mode for Orin NX
    • Speed: 35 FPS on Orin NX (vs 8 FPS FP32 PyTorch)

    5. Deployment and Integration

    • Runtime: TensorRT C++ API for low overhead
    • Preprocessing: GPU-accelerated resize and normalize using CUDA
    • Postprocessing: Efficient NMS implementation, confidence threshold tuning
    • Tracking: Simple IOU tracker for temporal smoothing of detections
    • Power: 8.2W average consumption at 35 FPS

    Best Practices Checklist

    Based on lessons learned from real-world projects:

    Data Practices

    • Spend 50% of project time on data - it's that important
    • Start small, iterate fast - collect 100 good annotations before scaling
    • Use data versioning (DVC, Git LFS) to track changes
    • Always keep a holdout test set that mimics production conditions
    • Annotate edge cases deliberately - they're worth 10x normal samples

    Model Development

    • Start with a pre-trained model and simple baseline
    • Use learning rate finders and systematic hyperparameter search
    • Validate with metrics that matter to your business (not just accuracy)
    • Implement early stopping to prevent overfitting
    • Test for bias - does performance vary across conditions?

    Optimization Strategies

    • Measure before optimizing - profile to find real bottlenecks
    • Consider the full pipeline - sometimes preprocessing is the bottleneck
    • Quantize early and often - catch accuracy issues sooner
    • Test on target hardware weekly, not just at the end
    • Keep an unoptimized "golden model" for comparison and debugging

    Deployment and Monitoring

    • Plan for model updates - how will you retrain and redeploy?
    • Implement logging and metrics collection in production
    • Set up alerts for data drift or performance degradation
    • Consider A/B testing for model updates in production
    • Have a fallback rule-based system for critical failures
